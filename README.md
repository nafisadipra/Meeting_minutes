# Grammatica — Local Meeting Assistant

Privacy-first meeting transcription with automatic speaker identification, live diarization, and AI-generated minutes. Everything runs on your machine — no cloud, no subscriptions.

---

## What It Does

- **Live transcription** via `faster-whisper` (Whisper base.en, CPU-optimised with int8)
- **Automatic speaker detection** — identifies who is speaking using ECAPA-TDNN voice embeddings; new voices are discovered mid-meeting without pre-enrollment
- **Voice enrollment** — optionally record a 5-second sample to give someone a real name instead of "Guest_01"
- **Meeting minutes** — structured Markdown summary generated by a local Ollama LLM when the meeting ends
- **Apple Silicon optimised** — CTranslate2 int8 on ARM NEON for Whisper; no GPU required

## Prerequisites

### System dependencies (install before Python packages)

**macOS**
```bash
brew install portaudio ffmpeg
```

**Ubuntu / Debian**
```bash
sudo apt-get install portaudio19-dev ffmpeg
```

**Windows**

Download and install:
- [PortAudio](http://www.portaudio.com/download.html) — or install via `conda install portaudio`
- [FFmpeg](https://ffmpeg.org/download.html) — add to PATH

> `portaudio` is required by `PyAudio`, which is the microphone backend used by `SpeechRecognition`. Without it the app cannot capture audio.

### Ollama (for meeting minutes)

Install from [ollama.com](https://ollama.com) and pull the model:
```bash
ollama pull llama3.2:3b
ollama serve          # runs on http://localhost:11434 by default
```

You can swap to any other model by setting `OLLAMA_MODEL` in `backend/.env`.

## Installation

### 1. Clone

```bash
git clone <repo-url>
cd Meeting_minutes
```

### 2. Backend

```bash
cd backend
```

Create and activate a virtual environment (recommended):
```bash
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
```

Install Python dependencies:
```bash
pip install -r requirements.txt
```

Install PyAudio (requires PortAudio to be installed first — see above):
```bash
pip install pyaudio
```

> **macOS troubleshooting**: if `pip install pyaudio` fails, try:
> ```bash
> pip install --global-option='build_ext' --global-option='-I/opt/homebrew/include' --global-option='-L/opt/homebrew/lib' pyaudio
> ```
> Or with conda: `conda install pyaudio`

Download the AI models (run once, requires internet, ~500 MB):
```bash
python download_models.py
```

This saves:
- `backend/models/whisper/base.en/` — Whisper speech-to-text
- `backend/models/speechbrain/spkrec-ecapa-voxceleb/` — speaker embedding model

### 3. Frontend

```bash
cd frontend
npm install
```

## Configuration

`backend/.env` controls all runtime paths and model settings:

```env
# Paths
MODELS_ROOT=./models
DATA_ROOT=./data
VOICE_PROFILES_DIR=voice_profiles
MINUTES_FILE=minutes.txt

# Whisper — CTranslate2 has no MPS/Metal backend.
# CPU + int8 uses ARM NEON on Apple Silicon and is already fast.
WHISPER_MODEL_PATH=./models/whisper/base.en
WHISPER_DEVICE=cpu
WHISPER_COMPUTE_TYPE=int8

# SpeechBrain — auto-selects CUDA > CPU (MPS unsupported by this model version)
SPEECHBRAIN_MODEL_PATH=./models/speechbrain/spkrec-ecapa-voxceleb

# Ollama
OLLAMA_URL=http://localhost:11434/api/generate
OLLAMA_MODEL=llama3.2:3b

# Flask
FLASK_PORT=5000
FLASK_DEBUG=False
```

To point the frontend at a different backend host, set in `frontend/.env.local`:
```env
NEXT_PUBLIC_API_URL=http://192.168.1.x:5000
```

## Voice Profiles

Saved as `.npy` files in `backend/data/voice_profiles/`.

- Enrolled profiles (named) persist across sessions
- Guests discovered during a session are offered for saving when the meeting ends
- Profiles are gitignored — they never leave your machine

---

## Data & Privacy

Nothing leaves the machine. The full pipeline:

```
Microphone → SpeechRecognition (local) → faster-whisper (local)
          → SpeechBrain embeddings (local) → Ollama (local)
```

No telemetry, no external API calls, no cloud storage.

## Troubleshooting

**`No module named 'pyaudio'`**
Install PortAudio first (see Prerequisites), then `pip install pyaudio`.

**`OSError: PortAudio library not found`**
PortAudio is not installed or not on the library path.
- macOS: `brew install portaudio`
- Linux: `sudo apt-get install portaudio19-dev`

**`Whisper model not found`**
Run `python download_models.py` from inside the `backend/` directory.

**Ollama not responding**
Make sure `ollama serve` is running. Minutes generation will show `[Ollama error: ...]` in the output file if it cannot connect. The transcript is still saved regardless.

**Speaker always shown as Guest_01**
The system needs at least ~1.5 seconds of clean audio per utterance to extract a reliable embedding. Background noise, overlapping speech, or a very quiet microphone reduce detection quality. Use a dedicated microphone and ensure the room is reasonably quiet.
